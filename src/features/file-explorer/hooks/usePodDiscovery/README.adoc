= usePodDiscovery

The application discovers the user's Pod storage root upon login and creates the app container (`my-solid-app/`)automatically if needed, with a retry handler for missing storage scenarios.

Discovery first attempts to retrieve the `pim:storage` triple from the user's profile. When unavailable, it falls back to a specification-based storage walk through the WebID resource, using `Link: rel="type"`headers to locate the root container and storage resources.

Used internally by `useDriveInitialization`.
Use it directly when you only need the storage root without navigation state or contacts.

== Returns

`appContainerUri`:: URI of the `my-solid-app/` container
`storageRootUri`:: Root URI of the user's Pod storage
`noStorageDetected`:: `true` when neither the profile's `pim:storage` triple nor the root-container walk yielded a storage root
`handleRetryStorage`:: Reloads the profile resource and resets discovery state; also called automatically after `storageRetryDelayMs`
`initialCurrentUri`:: Storage root URI to use as the navigation starting point
`initialBreadcrumbLabel`:: Translated label for the root breadcrumb entry
