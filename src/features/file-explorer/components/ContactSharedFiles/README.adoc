= ContactSharedFiles

Renders the resources a single contact shares with the viewer.
Extracted from `SharedWithMeSection` so the OneDrive Shared and People views can render the same list with their own filtering applied.

Reads the contact's shared catalog through `useSharedCatalog`, resolves the contact's display name from their Solid profile, and renders one read only `FileCard` per shared file plus collapsible `TypeFolder` groups for files the contact has only made available by class.

Returns nothing when the contact's catalog is not readable by the viewer.

== Props

[cols="1,3", options="header"]
|===
| Prop | Description

| `contactWebId` | WebID of the contact whose shares we are reading
| `viewerWebId` | WebID of the viewer. Used for ACL checks.
| `entryFilter` | Optional predicate applied to each catalog entry. SharedView and PeopleView use it to scope the list to a chip selection.
| `hideFromHeading` | Hides the "From {{name}}" header when a parent surface already shows the contact's name, for example the People detail view
|===
