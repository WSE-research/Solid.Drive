= RequestsView

Access requests view for the OneDrive layout.

OneDrive itself has no direct access requests surface, so this view borrows the rest of the layout's vocabulary: an elevated panel, the shared design tokens, and the same card, action, and empty state patterns used by SharedView and the file table.

The list reads its data from link:../../../../profile/contexts/RequestNotificationsContext/[RequestNotificationsContext], which is mounted once at the app root by link:../../../../../app/RequestNotificationsGate/[RequestNotificationsGate] and is fed live by the Solid Notifications WebSocket subscription. Until drive initialisation reports a storage root and catalog URI, the view renders a quiet "Connecting…" placeholder.

== Internal pieces

* `RequestsList` reads `requests, loading, error, busyMessageUri, approve, deny, selectedRequestId` from the notifications context, marks visible requests as seen on mount, and scrolls the selected request into view when the user deep-links from the bell dropdown.
* `RequestCard` renders one card per pending request with Approve and Deny actions.
