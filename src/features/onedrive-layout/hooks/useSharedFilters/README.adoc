= useSharedFilters

Filter state shared between the SharedView and the SharedView style toolbar inside PeopleView.
Holds a single selected chip id and a free text query that matches against contact display names and WebIDs.

The chip list itself is derived externally from observed catalog entries.
See `chipCatalog.deriveChipsFromEntries`.
This hook only owns the selected ids and the person query.
The matcher takes the chip list as input, so the predicate stays stateless.

Chip selection is single select.
Clicking a chip switches the filter to that chip alone.
Clicking it again clears the filter, returning to "All".

== Returns

[cols="1,3", options="header"]
|===
| Field | Description

| `selectedClasses` | Currently selected chip ids. Empty means "all".
| `toggleClass(chipId)` | Switches the active chip
| `resetClasses()` | Clears the type filter
| `personQuery` | Current person and name query
| `setPersonQuery(next)` | Updates the query
| `isActive` | `true` when at least one filter is active
| `matchesEntry(entry, chips)` | Predicate for a catalog entry given the current chip selection
| `matchesContact(name, webId)` | Predicate for a contact given the person query
|===
