= useLayoutPreference

Reads and persists the user's choice between the classic explorer and the OneDrive layout.

Defaults to `'classic'` so existing users see no change on first load.
A corrupted or missing stored value falls back to the same default.

Every setter dispatches a `solid-drive:layout-changed` event that all live hook instances listen for, so flipping the preference in one component immediately updates the rest.
The hook also listens for the cross tab `storage` event.

When `localStorage` is unavailable, for example in private browsing or when quota is exceeded, the in memory state still updates for the current session.
The cross instance event is skipped in that case so other live hooks do not overwrite their state from the stale stored value.

== Exports

`isLayout(value)`:: Type guard for the `Layout` union. Exported so call sites that receive untyped values can narrow with the same source of truth.
`useLayoutPreference()`:: Returns `[layout, setLayout]`.
