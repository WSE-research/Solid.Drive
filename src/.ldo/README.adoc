= .ldo

Auto generated by the LDO (Linked Data Objects) toolchain from the ShEx shapes in `../.shapes/`.
Do not edit these files directly, they will be overwritten on the next generation run.
To change a shape, edit the `.shex` file in `.shapes/` and rerun the generator.

== Contents

`solidProfile`:: Solid WebID profile (name, knows, storage, inbox, etc.)
`catalogEntry`:: DCAT catalog entry metadata (title, publisher, format, size, etc.)

== Regenerating

[source,bash]
----
npm run build:ldo
----
