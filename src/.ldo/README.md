# .ldo

## Overview

Auto generated by the LDO (Linked Data Objects) toolchain from the ShEx shapes in `../.shapes/`. Don't edit these files directly as they will be overwritten on the next generation run. To change the shape, edit the `.shex` file in `.shapes/` and rerun the generator.

## Contents

| Name | Description |
|---|---|
| `solidProfile` | Solid WebID profile (name, knows, storage, inbox, etc.) |
| `catalogEntry` | DCAT catalog entry metadata (title, publisher, format, size, etc.) |

## Usage

Run `npm run build:ldo` to regenerate after editing a `.shex` file.
