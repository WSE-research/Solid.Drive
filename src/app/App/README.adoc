= App

Root component that bootstraps the application.
Wraps everything in the Solid LDO provider, the notification provider, and a single `RequestNotificationsGate`, then picks the active shell based on auth state and the saved layout preference.

== Layout

[cols="1,2", options="header"]
|===
| Condition | Renders

| Always | `<BrowserRouter>` > `<BrowserSolidLdoProvider>` > `<NotificationProvider>` > `<RequestNotificationsGate>`
| Logged out | `<LandingPage />`
| Logged in, OneDrive layout | `<OneDriveLayout />`
| Logged in, Classic layout | `<Header />` and `<ClassicLayout />`
|===

The shell choice also honours session continuity: during the auth-restore window after a refresh, the previously logged-in shell stays mounted instead of flashing back to the LandingPage.
