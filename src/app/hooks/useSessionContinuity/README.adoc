= useSessionContinuity

Tells AppShell which shell to render while Solid auth is settling.

`useSolidAuth().session.isLoggedIn` lies for about a second after page load: it returns `false` while the provider is restoring a saved session or finishing an OIDC redirect.
Two things break without help here:

* Refreshing the page inside the OneDrive layout briefly flashes the logged-out shell.
* Coming back from the issuer with `?code&state` bounces the user back to the LandingPage for a moment.

This hook smooths over both cases.
It writes a `solid-drive.session-active` flag to `sessionStorage` whenever a session is live, so a refresh can tell that auth is restoring rather than starting fresh.
It also checks the URL on mount: if the OIDC callback params are there, it holds a short boot window (~2.5s) so the next layout gets a head start on pod discovery instead of mounting into an empty loading state.

== Returns

`assumeLoggedIn`:: `true` while a session is live, or being restored after a refresh. Flips to `false` once the user really logs out.
`isAuthenticating`:: `true` while the OIDC callback is being processed or the boot window is still open. AppShell shows the auth-callback skeleton while this is `true`.
